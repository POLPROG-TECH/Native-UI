# Theming

Native-UI is driven by a **single resolved theme object** that every
component consumes via hooks. You never touch individual components to
rebrand - you change the provider config and the whole tree updates
synchronously.

The theme system is built around three concepts:

1. **Presets** - curated accent + neutral pairings (7 built-in).
2. **Custom accent** - a single hex color that rebrands the whole app.
3. **Semantic tokens** - designer-facing aliases that decouple intent
   from raw values (`semantic.colors.danger` rather than `colors.error`).

## 2. Color presets

Each preset ships matched light **and** dark palettes plus a primary
accent. Switching preset is a one-prop change.

| Preset | Light accent | Dark accent | Use case |
|---|---|---|---|
| `default` | Indigo `#4F46E5` | Indigo `#818CF8` | General-purpose enterprise apps |
| `midnight` | Slate blue `#6366F1` | Slate `#A5B4FC` | Professional / corporate |
| `ocean` | Cyan `#0891B2` | Cyan `#22D3EE` | Fresh, modern utilities |
| `forest` | Emerald `#059669` | Emerald `#34D399` | Health, nature, sustainability |
| `sunset` | Amber `#D97706` | Amber `#FBBF24` | Warm consumer / lifestyle |
| `rose` | Pink `#E11D48` | Pink `#FB7185` | Fitness, community, media |
| `amoled` | White `#FFFFFF` | Pure white on `#000000` | OLED battery savings |

```tsx
<NativeUIProvider config={{ preset: 'forest', colorMode: 'system' }}>
```

> Switch presets **live** using the **Preset** toolbar at the top of
> any story in this Storybook to see the impact across every component.

## 3. Color modes

```tsx
<NativeUIProvider config={{ colorMode: 'system' }}>
```

- `'light'` / `'dark'` - forced, useful for settings screens that let
  users override the OS preference.
- `'system'` - follows `useColorScheme()`; re-renders automatically when
  the user flips the OS toggle.

Each preset resolves to a full `ColorScheme` of 25+ semantic tokens so
you never hand-pick a hex.

### What you get (resolved scheme)

<details>
<summary>Surfaces</summary>

- `background` - App root background.
- `surface` - Card / sheet / list background.
- `surfaceElevated` - Modals, popovers.
- `surfaceSecondary` - Grouped content (settings sections).
- `overlay` - Semi-transparent backdrop behind modals / sheets.

</details>

<details>
<summary>Text</summary>

- `textPrimary` - Body + headings.
- `textSecondary` - Captions, helper text.
- `textTertiary` - Muted / placeholder.
- `textInverse` - Text that sits on colored backgrounds.
- `textOnPrimary` - Text on the accent color (auto WCAG-picked).

</details>

<details>
<summary>Semantic state colors</summary>

- `primary` / `primaryLight` / `primaryDark`
- `success` / `successLight`
- `warning` / `warningLight`
- `error` / `errorLight`
- `info` / `infoLight`

Each `*Light` variant is pre-tinted for badges, callouts and tinted
surfaces (~9 % alpha over the base).

</details>

<details>
<summary>System</summary>

- `border` / `borderLight` - Hairlines at 1px / 0.5px.
- `divider` - List separators.
- `shadow` - Elevation colour (iOS shadow, Android elevation).

</details>

## 4. Custom accent (brand override)

One hex color. Done.

```tsx
<NativeUIProvider config={{ customAccent: '#7C3AED', colorMode: 'system' }}>
```

Internally `customAccent` flows through `tintColor()` and the contrast
picker so:

- `primary` = your hex
- `primaryLight` = your hex at ~15 % alpha (for tinted surfaces)
- `primaryDark` = your hex luminance-shifted for pressed states
- `textOnPrimary` = auto-picked black/white for WCAG AA on your hex

> **Trade-off.** `customAccent` overrides **only** the primary family.
> `success` / `error` / neutrals stay on the preset. If you need
> full brand-level overrides, use the `semantic` prop instead (next
> section).

## 5. Semantic token overrides

For a complete rebrand, pass a partial `semantic` object. Anything you
omit falls back to the preset defaults, so you only override what you
care about.

```tsx
<NativeUIProvider
  config={{
    preset: 'default',
    semantic: {
      colors: {
        brand: '#0066CC',
        brandMuted: '#DCE8F4',
        danger: '#D32F2F',
      },
      spacing: { screenGutter: 20 }, // default is 16
    },
  }}
>
```

Components that consume semantic aliases (`useSemantic()`) switch
instantly. Good candidates for per-app override: `brand`, `danger`,
`successToneBg`, `screenGutter`.

## 6. Accessibility toggles

All four toggles compose; every component responds without any prop
drilling.

| Config | Effect |
|---|---|
| `fontSize: 'large'` | +4 px across every type variant, matching iOS "Larger Text". |
| `highContrast: true` | Promotes `textSecondary` to `textPrimary` weight / opacity. |
| `reduceAnimations: true` | `PressableScale` and `Skeleton` skip animations; `BottomSheet` / `Toast` use instant transitions. |
| `fontColor: 'warm'` / `'cool'` | Tints text tokens by ±2° on the color wheel for eye strain. |

```tsx
<NativeUIProvider
  config={{
    fontSize: 'large',
    highContrast: true,
    reduceAnimations: true,
  }}
>
```

> **Pair with OS preferences.** In real apps, drive these values from
> `AccessibilityInfo.isReduceMotionEnabled()` and
> `PixelRatio.getFontScale()` rather than hard-coding them.

## 7. Typography density

| Density | Body | Use case |
|---|---|---|
| `regular` | 17/22 (iOS-default) | Marketing, primary flows, read-once screens |
| `compact` | 15/20 | Settings bundles, transaction lists, data-dense screens |

```tsx
<NativeUIProvider config={{ typography: 'compact' }}>
```

Compact recovers ~12 % vertical real estate on settings screens without
information loss. See **Components / SettingsRow** for the side-by-side
comparison.

## 8. Consuming theme in custom components

```tsx
import { useTheme, useColors, useSemantic } from '@polprog/native-ui';

// Heaviest: full theme object (re-renders on any token change)
const theme = useTheme();
theme.colors.primary;
theme.typography.h1;
theme.spacing.lg;
theme.borderRadius.md;
theme.elevation.md;

// Medium: just the color scheme
const colors = useColors();

// Lightest: designer-facing aliases (brand, danger, screenGutter, ...)
const semantic = useSemantic();
```

Pick the smallest hook. They are backed by separate React Contexts so
your custom component only re-renders when the slice it reads actually
changes.

## 9. Brand-specific theme recipe

```tsx
const BRAND_CONFIG: NativeUIConfig = {
  colorMode: 'system',
  preset: 'default',
  customAccent: '#00E67A',    // brand accent
  fontColor: 'default',
  fontSize: 'default',
  highContrast: false,
  reduceAnimations: false,
  typography: 'regular',
  semantic: {
    colors: {
      brand: '#00E67A',
      brandMuted: '#DCFCE7',
    },
  },
};

export default function App() {
  return (
    <NativeUIProvider config={BRAND_CONFIG}>
      <Root />
    </NativeUIProvider>
  );
}
```

> **Do not** spread theme config through app state if you can avoid it.
> Pin it to a module constant - React will skip the reconciliation cost
> of a new config object on every render.
