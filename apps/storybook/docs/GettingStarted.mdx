# Getting Started

A one-page, production-grade checklist to get `@polprog/native-ui`
running in your React Native app - from fresh install to first themed
screen in under five minutes.

> **Who this is for.** Teams that already have a React Native project
> (bare RN or Expo SDK 50+) and want a drop-in design system that is
> token-driven, accessible by default, and production-ready.

---

## 1. Prerequisites

| Requirement | Minimum | Notes |
|---|---|---|
| React Native | **0.72** | 0.76+ recommended for New Architecture |
| React | **18.0** | Required for automatic JSX + concurrent features |
| TypeScript | **5.0** | Types ship inside the package; `skipLibCheck` optional |
| `react-native-reanimated` | **3.0** | Powers `PressableScale`, `BottomSheet`, `Skeleton` |
| `react-native-safe-area-context` | **4.0** | Required by `ScreenContainer` + `BottomTabBar` |

Native-UI is framework-agnostic: it works under the React Native CLI,
Expo (managed or bare), and - for documentation purposes - under
`react-native-web` inside Storybook / Vite.

---

## 2. Install

```bash
# Package + peer deps
npm install @polprog/native-ui react-native-reanimated react-native-safe-area-context

# Pod install for iOS (bare RN only)
cd ios && pod install && cd ..
```

> **Reanimated setup.** If this is a brand-new project, follow the
> Reanimated
> [installation guide](https://docs.swmansion.com/react-native-reanimated/docs/fundamentals/installation)
> to add the Babel plugin - otherwise `PressableScale` will throw on
> first render.

---

## 3. Wrap your app

A single provider is the whole setup. Everything downstream reads theme,
typography, spacing and motion tokens from it.

```tsx
import { NativeUIProvider } from '@polprog/native-ui';
import { SafeAreaProvider } from 'react-native-safe-area-context';

export default function App() {
  return (
    <SafeAreaProvider>
      <NativeUIProvider
        config={{
          colorMode: 'system',      // 'light' | 'dark' | 'system'
          preset: 'default',         // 7 built-in presets + customAccent
          fontSize: 'default',       // a11y scale: default | medium | large
          reduceAnimations: false,   // respect OS "Reduce Motion"
        }}
      >
        <RootNavigator />
      </NativeUIProvider>
    </SafeAreaProvider>
  );
}
```

> **Order matters.** `SafeAreaProvider` **must** wrap `NativeUIProvider`
> so `ScreenContainer` and `BottomTabBar` can read insets synchronously
> on first render - otherwise you will see a one-frame layout jump on
> iPhone Pro / Dynamic Island devices.

## 4. First screen

```tsx
import {
  Button,
  Card,
  Heading,
  ScreenContainer,
  Text,
  VStack,
} from '@polprog/native-ui';

export function HomeScreen() {
  return (
    <ScreenContainer>
      <VStack gap="lg" style={{ padding: 16 }}>
        <Heading level={1}>Welcome back</Heading>
        <Card>
          <VStack gap="md">
            <Text variant="body">
              This card inherits surface, text and spacing tokens from the
              provider. No explicit colors or font sizes anywhere.
            </Text>
            <Button title="Continue" onPress={() => {}} />
          </VStack>
        </Card>
      </VStack>
    </ScreenContainer>
  );
}
```

That is a full screen: themed surface, safe-area handling, semantic
typography, token-driven spacing, accessible pressable - zero setup
beyond step 3.

## 5. Haptic feedback (optional)

Interactive primitives (`Button`, `PressableScale`, `Toast`) call into a
pluggable haptics interface. Wire it once at app boot, then every
component benefits.

```tsx
import { configureHaptics } from '@polprog/native-ui';
import ReactNativeHapticFeedback from 'react-native-haptic-feedback';

configureHaptics({
  light: () => ReactNativeHapticFeedback.trigger('impactLight'),
  medium: () => ReactNativeHapticFeedback.trigger('impactMedium'),
  success: () => ReactNativeHapticFeedback.trigger('notificationSuccess'),
  warning: () => ReactNativeHapticFeedback.trigger('notificationWarning'),
  error: () => ReactNativeHapticFeedback.trigger('notificationError'),
});
```

> **Graceful degradation.** If you never call `configureHaptics`, every
> component silently skips haptics - no crashes, no warnings, no bundle
> bloat. Perfect for Expo Go dev.

## 6. Consume tokens in custom components

The provider exposes the full resolved theme through three tiny hooks:

```tsx
import { useTheme, useColors, useSemantic } from '@polprog/native-ui';

function CustomCard() {
  const theme = useTheme();        // colors + typography + spacing + radii + ...
  const colors = useColors();       // just the color scheme (cheaper)
  const semantic = useSemantic();   // designer-facing aliases

  return (
    <View
      style={{
        backgroundColor: colors.surface,
        padding: theme.spacing.lg,
        borderRadius: theme.borderRadius.md,
      }}
    >
      <Text style={theme.typography.h2}>Themed</Text>
    </View>
  );
}
```

Pick the smallest hook that covers your need - each one only re-renders
its consumers when the values it reads actually change.

## 7. Provider configuration reference

| Prop | Type | Default | Description |
|---|---|---|---|
| `colorMode` | `'light'` / `'dark'` / `'system'` | `'system'` | Follows `useColorScheme()` when `'system'` |
| `preset` | `ThemePreset` | `'default'` | One of 7 built-in color presets |
| `customAccent` | `string` or `null` | `null` | Hex color that overrides `primary`/`primaryLight`/`primaryDark` |
| `fontColor` | `'default'` / `'warm'` / `'cool'` | `'default'` | Tints text tokens (accessibility-friendly) |
| `fontSize` | `'default'` / `'medium'` / `'large'` | `'default'` | Global a11y scale (+0 / +2 / +4 px) |
| `highContrast` | `boolean` | `false` | Strengthens text-on-surface contrast |
| `reduceAnimations` | `boolean` | `false` | Disables springs / shimmer / scale animations |
| `typography` | `'regular'` / `'compact'` / `Typography` | `'regular'` | Density preset (compact saves ~12 % vertical) |
| `fontFamilies` | `FontFamilies` | system | Override to ship your brand font |
| `semantic` | `SemanticTokens` | defaults | Override semantic color / spacing / type tokens |

See **Theming** for a deep dive on each of these.

## 8. Monorepo setup

For teams shipping multiple apps from one repo:

### Metro config (`metro.config.js`)

```js
const path = require('path');
const { getDefaultConfig } = require('@react-native/metro-config');

const config = getDefaultConfig(__dirname);
config.watchFolders = [
  path.resolve(__dirname, '../../packages/native-ui'),
];
module.exports = config;
```

### TypeScript path mapping (`tsconfig.json`)

```json
{
  "compilerOptions": {
    "paths": {
      "@polprog/native-ui": ["../../packages/native-ui/src/index"]
    }
  }
}
```

> **Symlink vs file dep.** For CI reliability, prefer `"file:../../packages/native-ui"` in
> `package.json` over `npm link` - it survives `npm ci` and locks the
> exact source tree used by the build.
